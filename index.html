关于 Agent Skill的思考

+

Agent Skill为什么突然火了？他有什么能力？和MCP有什么区别？

Agent Skill MCP 工具

Agent Skill 的本质可以理解为 AI 模型随时翻阅的"说明文档"，让模型知道在特定任务中该如何工作，无需每次都重复粘贴指令。

渐进式披露机制是他的核心优势，主要是采用三层加载结构来节省 Token。首先是元数据层，他是始终加载的，其中信息包含了所有 Skill 的名称和描述，但是这部分相对而言，就可以类比MCP中的工具和工具描述，不会很长，只是充当模型翻阅目录时的索引。然后是指令层，我个人理解这里的指令层就是预置的prompt，我们只有只有被选中的 Skill 的完整指令才会加载，通过这种渐进式大大的降低了找工具并执行的消耗。最后是资源层，是按需中的按需，比如我这个 Skill 里面写了很多代码，我们如果在指令层写清楚了希望当某时执行这块代码，只会按需执行，这部分写在Reference 文件和 Script 脚本中。

Reference：条件触发的参考文档，会被读取并占用上下文（例如：只有会议涉及费用时才加载财务手册）

Script：可执行的代码脚本，只运行不读取，几乎不占用上下文（例如：自动上传文件到服务器）

Anthropic 在官方文档中用一句话精准定义了两者的关系：

"MCP connects Claude to data, Skills teach Claude what to do with that data."

MCP：连接 Claude 与数据源

Agent Skill：教会 Claude 如何处理这些数据

Agent Skill 代表了 AI Agent 领域的一次重要创新，它通过渐进式披露机制解决了大模型应用中的 Token 消耗痛点，通过模块化设计提升了可维护性，通过开放标准构建了跨平台生态。

随着 Agent Skill 成为开放标准，预期将会看到：更丰富的社区 Skill 库、更多工具和平台的支持、企业级 Skill 管理平台、与 MCP 更深度的整合。

Agent Skill 正在从一个产品特性演变为 AI Agent 领域的基础设施，值得每一个 AI 应用开发者深入了解和实践。
